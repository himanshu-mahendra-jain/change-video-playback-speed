# Change Video Playback Speed

A lightweight browser extension for quickly changing video and audio playback speed with a toolbar click or keyboard shortcut.

The extension is designed to be simple, fast, private, and predictable. It has no popup, no floating controller, no settings page, no analytics, and no broad website access.

## Supported Browsers

The extension supports:

* Google Chrome
* Microsoft Edge
* Mozilla Firefox

Chrome and Edge use the Chromium manifest. Firefox uses a separate manifest for browser-specific background execution, keyboard shortcuts, and add-on metadata.

The JavaScript source code is shared across all supported browsers.

## Features

* Cycle playback speed with one toolbar click
* Use a keyboard shortcut to change speed
* Synchronize video and audio elements to the same playback speed
* Support media in accessible iframes
* Show a temporary on-page speed indicator
* Show the selected speed in the extension toolbar badge
* Handle rapid repeated actions sequentially per tab
* No persistent background monitoring
* No storage or browsing-history collection
* No network requests
* No analytics or tracking

## Playback Speed Cycle

The extension cycles through the following speeds:

```text
1x → 1.2x → 1.5x → 2x → 1x
```

If the current playback rate is not one of the configured values, the extension selects the next higher configured speed.

Examples:

```text
1.1x  → 1.2x
1.3x  → 1.5x
1.75x → 2x
2.5x  → 1x
```

## Usage

### Toolbar

Click the extension icon to move to the next playback speed.

### Keyboard Shortcut

The extension includes a default keyboard shortcut for quickly cycling through the available playback speeds.

#### Chrome and Edge

| Operating System | Shortcut |
| ---------------- | -------- |
| Windows | `Alt+S` |
| Linux | `Alt+S` |
| ChromeOS | `Alt+S` |
| macOS | `Command+Shift+Period` |

#### Firefox

| Operating System | Shortcut |
| ---------------- | -------- |
| Windows | `Ctrl+Shift+Period` |
| Linux | `Ctrl+Shift+Period` |
| macOS | `Command+Shift+Period` |

The keyboard shortcut performs exactly the same action as clicking the extension icon.

Users can change the shortcut at any time through their browser's extension keyboard shortcut settings.

### `manifest.json`

Defines the extension metadata, permissions, service worker, toolbar action, and keyboard command for Chrome and Edge. The service worker uses ES modules.

### `manifest-firefox.json`

Defines the Firefox-specific extension configuration while using the same permissions, source files, and functionality.

### `src/background.js`

Handles toolbar clicks, keyboard shortcut commands, active-tab lookup for keyboard commands, and per-tab operation queues.

The per-tab queue ensures rapid actions are processed sequentially within the same tab while allowing different tabs to operate independently.

### `src/playback.js`

Coordinates the two-pass playback-speed workflow: first inspecting accessible frames, then applying one synchronized speed to media across accessible frames.

### `src/page.js`

Contains self-contained functions executed in webpage contexts to inspect media state and apply playback speed.

### `options/`

Contains a placeholder only. The extension does not expose a user-facing options page or settings UI.

### `store-assets/`

Contains browser-store listing text and promotional images. These files are repository assets and are not included in browser distribution packages.

## Iframe Handling

The extension uses a tab-level two-pass strategy for iframe support. The speed is calculated once and then applied consistently across accessible frames.

Some embedded frames may not be accessible because of browser security restrictions and the extension's intentionally limited permission model.

## Toolbar Badge

After changing playback speed, the extension displays the selected value in the toolbar badge.

Possible badge values are:

```text
1
1.2
1.5
2
```

The badge is tab-specific.

## Speed Indicator

Every successful playback-speed change displays a temporary indicator in the center of the page. It appears only in the top frame, does not intercept pointer events, cannot be selected, uses a very high stacking order, and automatically disappears after one second.

## Permissions

The extension requests only:

```json
{
  "permissions": [
    "activeTab",
    "scripting"
  ]
}
```

### `activeTab`

Provides temporary access to the active tab after the user invokes the extension. The extension does not request permanent access to all websites.

### `scripting`

Allows the extension to execute the media detection and playback-speed functions in the user-invoked tab and accessible frames.

## Privacy

The extension:

* Does not collect user data
* Does not collect browsing history
* Does not use analytics
* Does not use tracking
* Does not make network requests
* Does not store playback history
* Does not store per-site preferences
* Does not use remote code
* Does not load external JavaScript
* Does not request broad host permissions

All playback-speed operations happen locally in the browser.

## Performance

The extension is event-driven and performs no continuous work while idle.

There is no polling, `setInterval`, `MutationObserver`, persistent content script, automatic media monitoring, or background network activity.

Work occurs only after a toolbar click or keyboard shortcut.

## Security Design

The extension follows a minimal-capability approach and intentionally avoids broad host permissions, remote code, external dependencies, analytics SDKs, advertising SDKs, remote configuration, web-accessible resources, unnecessary message listeners, native messaging, and persistent page monitoring.

Playback speeds are validated before being applied to media elements.

## Build

Download `build.sh` from the [Web Extensions Common Build Script repository](https://github.com/himanshu-mahendra-jain/web-extensions-common-build-script) and place it in the extension project's root directory.

The build script is maintained separately so the same packaging process can be reused across multiple browser extension projects.

### Prerequisites

The build process requires:

* Bash
* `zip`

Check whether `zip` is installed:

```bash
zip -v
```

If it is not installed, install it using your system package manager.

Examples:

```text
Debian/Ubuntu: sudo apt install zip
```

### Download the Build Script

Download the shared `build.sh` script from the **Web Extensions Common Build Script** repository and place it in the extension project's root directory.

After adding the script, the local project structure should look like this:

```text
change-video-playback-speed/
├── icons/
├── options/
├── src/
├── store-assets/
├── build.sh
├── manifest.json
├── manifest-firefox.json
├── LICENSE
└── README.md
```

The downloaded build script is a development tool and does not need to be included in browser distribution packages.

### Run the Build

Make the build script executable:

```bash
chmod +x ./build.sh
```

Run:

```bash
./build.sh
```

The script:

* Reads the extension name and version from the manifest files
* Creates a clean `dist/` directory
* Packages Chrome using `manifest.json`
* Packages Edge using `manifest.json`
* Packages Firefox using `manifest-firefox.json` as `manifest.json` inside the Firefox ZIP
* Includes the shared `src/` and `icons/` directories
* Creates separate ZIP files for each browser

For details about the shared build process, updates, and usage instructions, see the [Web Extensions Common Build Script repository](https://github.com/himanshu-mahendra-jain/web-extensions-common-build-script).

Example output:

```text
dist/
├── change-video-playback-speed-chrome-v1.0.2.zip
├── change-video-playback-speed-edge-v1.0.2.zip
└── change-video-playback-speed-firefox-v1.0.2.zip
```

The ZIP archives contain only the files required to run the extension. Repository documentation and store-listing assets are not included in the distribution packages.

## Known Limitations

* Some browser-protected pages do not allow script injection.
* Some embedded cross-origin players may not be accessible under the extension's limited permission model.
* Websites may independently override playback speed after the extension applies it.
* The extension intentionally does not monitor pages continuously to reapply playback speed.
* Mobile browser support depends on the browser and platform's extension capabilities.

## Design Philosophy

Change Video Playback Speed is intentionally focused on one task:

> Change playback speed quickly, without menus, overlays, configuration, tracking, or permanent website access.

The extension favors explicit user actions, limited permissions, predictable behavior, and minimal background activity over automatic playback management.
